Модуль 1. Введение в ОС Astra Linux

Данный модуль является первым в курсе о базовом администрировании операционной системы (ОС) Astra Linux Special Edition 1.8 с использованием графических инструментов.

История операционной системы Astra Linux

Astra Linux — операционная система специального назначения на базе ядра Linux, была создана для нужд органов государственного и военного управления и других учреждений, которые работают с информацией ограниченного доступа.

  • Допустимый уровень защищённости - определяется приобретённой лицензией

В данном модуле рассматривается процесс установки операционной системы Astra Linux.

Процесс установки

Установка Astra Linux включает следующие основные этапы:

  1. Подготовка загрузочного носителя
  2. Загрузка с установочного носителя
  3. Выбор языка и раскладки клавиатуры
  4. Разметка дискового пространства
  5. Настройка параметров установки
  6. Создание учетной записи пользователя
  7. Завершение установки и перезагрузка

Видеоуроки

Для изучения материала доступны видеоуроки:

  • Видеоурок 1
  • Видеоурок 2

Требования к системе

Минимальные системные требования для установки Astra Linux Special Edition 1.8:

Модуль 3. Процесс загрузки ОС

Данный модуль содержит основные сведения о стадиях загрузки ОС. Вы ознакомитесь с загрузчиком ОС, его функциями и основными возможностями настройки.

Порядок загрузки ОС

  1. Процессор выполняет код BIOS/UEFI
  2. Загружается программа-загрузчик (GRUB2)
  3. GRUB2 загружает ядро Linux и образ initramfs
  4. Ядро инициализирует важные функции
  5. Монтируется образ initramfs и запускается процесс init
  6. Загружаются драйверы и монтируется системный раздел
  7. Процесс init замещается службой systemd
  8. Systemd запускает службы и графическую подсистему

Загрузчик GRUB2

GRUB2 (GRand Unified Bootloader) - это современный загрузчик, используемый в системах семейства Linux.

Основные возможности GRUB2

  • Загрузка нескольких операционных систем
  • Модульная структура
  • Поддержка различных файловых систем
  • Возможность редактирования параметров загрузки

Systemd

Systemd - это система инициализации, "корневой" родитель всех процессов пользовательского пространства.

Модуль 4. Управление программным обеспечением

ОС Astra Linux сразу после установки имеет большой набор предустановленного программного обеспечения. Однако для решения специализированных задач может потребоваться установка новых программ или обновление существующих.

Программные пакеты

Программы для ОС Linux распространяются в программных пакетах. В Astra Linux используются пакеты формата .deb.

Репозитории

Все программы и обновления ОС располагаются в специальных централизованных хранилищах - репозиториях.

Основные репозитории Astra Linux

  • Основной репозиторий (main) - входит в поставочный комплект дистрибутива, сертифицируется
  • Расширенный репозиторий (extended) - содержит дополнительное прикладное и системное ПО

Менеджер пакетов Synaptic

Synaptic - это графический интерфейс к программе управления пакетами APT.

Основные возможности Synaptic

  • Поиск пакетов по разделам, состоянию, названию
  • Установка, удаление пакетов по выбору пользователя
  • Автоматический выбор для установки всех пакетов, имеющих обновления
  • Возможность работы с пакетами, установленными с ошибками
  • Встроенный редактор списка источников пакетов

Модуль 5. Управление файловыми системами

В данном модуле рассматриваются принципы именования физических дисков и логических разделов в Astra Linux.

Именование файлов дисковых устройств

  • SCSI, SATA: /dev/sd*
  • Параллельные интерфейсы: /dev/hd*
  • NVMe SSD: /dev/nvme#

Поддерживаемые типы ФС в Astra Linux

"Родные" для Linux ФС - ext2/ext3/ext4. Также поддерживаются XFS, BtrFS, iso9660, udf.

Редактор разделов GParted

GParted - это мощный инструмент для работы с разделами жёсткого диска. Он позволяет создавать, удалять, перемещать и изменять размеры разделов без потери данных.

Форматирование внешнего носителя

Программа "Форматирование внешнего носителя" предназначена для удаления данных и форматирования внешнего носителя и его разделов.

Модуль 6. Работа с файлами в ОС Astra Linux

В данном модуле рассматривается логическая структура файловой системы ОС Astra Linux, а также назначение её основных папок.

Логическая структура файловой системы

В операционной системе Astra Linux используется единое файловое пространство в виде перевёрнутого дерева с корнем (root) вверху.

Типы файлов

  • Регулярные файлы
  • Каталоги
  • Символические ссылки
  • Файлы устройств
  • Локальные сокеты и именованные каналы

Стандарт иерархии ФС (FHS)

Основные каталоги файловой системы:

Каталог Назначение
/ Корневой каталог
/bin Основные программы пользователя
/boot Загрузочные файлы
/etc Общесистемные конфигурационные файлы
/home Домашние каталоги пользователей
/var Изменяемые файлы

Менеджер файлов

Менеджер файлов позволяет осуществлять базовые операции с файлами и папками: создавать, удалять, копировать, переносить, переименовывать.

Модуль 7. Процессы в Linux

В операционной системе постоянно работает большое количество различных процессов. В некоторых случаях вам может понадобиться найти зависший процесс и корректно его завершить.

Программа, процесс, потоки

Программа - исполняемый файл, который предназначен для создания процесса.

Процесс - экземпляр исполняющейся программы.

Мониторинг процессов

Мониторинг процессов может быть выполнен с помощью утилиты "Системный монитор".

Системный монитор: использование процессора и памяти

Системный монитор: общий вид с таблицей процессов

Системный монитор: меню файла и настройки

Системный монитор: свойства вкладки системы

Системный монитор: мониторинг числа процессов

Сигналы

С помощью системного монитора пользователь может отправить управляющий сигнал выбранному процессу:

  • Приостановить (STOP)
  • Возобновить (CONT)
  • Перечитать конфигурацию (HUP)
  • Прервать (INT)
  • Завершить (TERM)
  • Уничтожить (KILL)

Журналы

Для работы с журналами в Astra Linux Special Edition предусмотрены специализированные программы.

Модуль 8. Управление учётными записями пользователей и групп

Работа с учётными записями пользователей и групп пользователей является неотъемлемой частью администрирования и обеспечения информационной безопасности компьютеров и серверов.

Учётные записи

В Linux предусмотрено три типа пользователей:

  1. Суперпользователь (root)
  2. Обычные пользователи
  3. Системные пользователи

Параметры учётных записей

Информация об учётных записях хранится в файле /etc/passwd. Каждая строка в файле соответствует одной учётной записи.

Управление пользователями и группами

Управление учётными записями и группами в Astra Linux осуществляется в программе "Параметры системы".

Политики учётных записей

Управление политиками учётных записей также осуществляется в программе "Параметры системы".

Модуль 9. Настройка сети в Astra Linux

В большинстве случаев пользователям необходимо наличие подключение компьютера к локальной сети и (или) сети Интернет.

Сетевые интерфейсы

Сетевой интерфейс - программное соответствие физического или виртуального сетевого устройства, предназначенное для передачи данных между процессами через компьютерную сеть.

NetworkManager

Для настройки сетевых интерфейсов используется программа NetworkManager.

Апплет NetworkManager

Быстро получить доступ к информации о соединениях, а также перейти к их настройкам позволяет апплет NetworkManager, расположенный в области уведомлений панели задач.

Подключение сетевых ресурсов

Astra Linux SE обеспечивает возможность подключения к сетевому оборудованию и сетевым файловых хранилищам.

Подключение к принтерам

Подключение и настройка принтера в Astra Linux выполняются с помощью менеджера печати Fly.
